# 07 - UI Design System

---

# Purpose

This document defines the visual language of every user interface shown throughout the portfolio.

The interface should never resemble a traditional website.

It should always feel like it belongs inside the game world.

Every interface should appear to originate from the object being interacted with.

---

# Design Philosophy

The world is always the primary focus.

UI is secondary.

The player explores the environment first.

Information appears only when interacting with objects.

The room should remain visible whenever possible.

Avoid replacing the entire screen.

---

# General Rules

Avoid

- Webpage layouts
- Hero sections
- Navigation bars
- Sidebars
- Card grids
- Bootstrap style components
- Material Design interfaces

Prefer

- Floating windows
- Physical interfaces
- Object based interaction
- Minimal overlays
- Cinematic presentation
