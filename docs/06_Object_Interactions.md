# 06 - Object Interactions

---

# Purpose

This document defines the behaviour of every interactive object inside the portfolio.

Every object represents one portfolio section.

Every object has its own identity.

Every object has its own interaction sequence.

No two interactions should feel identical.

The object itself should become the interface.

---

# General Rules

Every interaction follows this structure.

Idle

↓

Discovery

↓

Interaction Prompt

↓

Camera Movement

↓

Object Animation

↓

Portfolio Interface

↓

Exit

↓

Return To Idle

---

# PROJECTOR

Portfolio Section

Projects

Purpose

Display all software projects.

---

Idle State

Projector is powered off.

Screen is rolled up.

A small standby LED glows softly.

---

Discovery

The projector sits beside a work desk.

It immediately attracts attention because it is the largest object in the workspace.

---

Interaction Prompt

Press E to Use Projector

---

Interaction Animation

The projector powers on.

Projector fan begins spinning.

The projector screen slowly rolls down.

A startup animation appears.

The room lighting becomes slightly dimmer.

The projected light becomes brighter.

---

Camera Behaviour

The camera walks naturally toward the projector.

Stops slightly behind the desk.

Looks directly at the screen.

No sudden movement.

---

Portfolio UI

The projector displays

Projects

↓

Project 1

Project 2

Project 3

...

Selecting a project opens

Project Name

Description

Tech Stack

Date

Platform

GitHub Link

Live Demo

Placeholder Image

---

Exit

Project closes.

Project list disappears.

Projector powers off.

Screen rolls upward.

Camera returns.

Movement restored.
